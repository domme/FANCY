# FANCY Engine — Shared Code Style Guide

**This is a git submodule of PathTracer.** Code style and architecture guidelines are centralized in the **parent project's** `.copilot/copilot-instructions.md` to ensure consistency across both the engine (FANCY/) and the application (PathTracer/).

When working on FANCY engine code:
1. Always refer to the **root-level** `../../.copilot/copilot-instructions.md`
2. Follow the same naming conventions, spacing rules, and formatting guidelines as the PathTracer application
3. Use the same `.clang-format` from the root directory

## Quick Links

- **Code Style Guide:** `../../.copilot/copilot-instructions.md`
- **Clang Format Config:** `../../.clang-format`

All architecture patterns, naming conventions, and development practices described in that guide apply equally to FANCY engine code.
